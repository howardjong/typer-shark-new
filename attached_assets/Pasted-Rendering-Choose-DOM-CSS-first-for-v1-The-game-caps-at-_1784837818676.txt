Rendering: Choose DOM/CSS-first for v1. The game caps at 3–5 active targets, so performance headroom does not justify a Canvas/DOM synchronization layer. Keep every target and its label as real DOM; drive movement from the central requestAnimationFrame game loop using transform, and reserve CSS animation for small visual effects. Keep React renders event-driven rather than re-rendering the whole scene every frame. Use CSS and small inline SVG for the original block-world background and creatures. Do not add Canvas unless profiling demonstrates a real bottleneck.
Audio: Choose (a). Use Web Audio API–generated, gentle original effects plus an optional subtle procedural ambient loop—no bundled audio files in v1. Audio must begin only after a user gesture, have separate effects/music controls, support captions, and fail silently without affecting play.
Testing: Use Vitest with React Testing Library where UI interaction needs coverage. Keep the deterministic game engine as plain TypeScript with focused unit tests. Add one lightweight browser smoke test if Replit’s testing environment supports it; do not introduce Jest.
State management: Use a dependency-free, discriminated-union TypeScript state machine with a pure event reducer and explicit guards. Keep simulation state outside React UI state. XState is unnecessary for this bounded state flow and would add maintenance.
Visual design: You are free to design it entirely from scratch within the clean-room constraints. No mood board or reference images are needed. Prioritize a bright, original underwater block-world, strong readable labels, calm motion, and friendly creatures; do not use Minecraft or Typer Shark visual assets, UI conventions, names, or recognisable designs.
Deployment cost: I understand that this must use Autoscale because Agent-created apps cannot use Static Deployment. Please use the smallest practical machine, start with a maximum of one instance, add no database/background worker, and surface the selected deployment settings and any cost estimate before the final publish action.
